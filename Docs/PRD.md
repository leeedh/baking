## Phase 1: Overview

### 1.1 한 줄 요약
**해외(대만 우선)·국내 수강생** 모두가 **쉽게 결제하고 안전하게 시청**할 수 있는 **다국어 베이킹 VOD 클래스 플랫폼**.

### 1.2 문제 정의
베이킹 클래스를 온라인으로 판매하려 하지만 두 가지 벽이 있다. 첫째, 해외 수강생은 **결제 수단과 언어 장벽**으로 결제 직전에 이탈한다. 둘째, VOD 영상은 **다운로드·재배포에 취약**해, 핵심 콘텐츠가 유출되면 수익 기반 자체가 무너진다. 기존 국내 강의 플랫폼은 해외 결제 접근성, 다국어, 영상 보안을 **동시에** 만족시키기 어렵다.

### 1.3 솔루션 요약
다국어 UI + 외국인 친화 결제 + 서명 기반 보안 스트리밍을 처음부터 내장한 VOD 클래스 플랫폼. **핵심 차별점**: 해외(대만 우선) 결제 접근성과 영상 유출 방지를 MVP 단계부터 기본 탑재해, 글로벌 판매와 콘텐츠 자산 보호를 동시에 달성한다.

**스택**: Next.js 15 + Supabase + Mux + 해외 PG (확정)

### 1.4 목표 (Goals)
| ID | 목표 | 측정 가능한 의미 |
|----|------|------------------|
| PRD-G-01 | 해외 수강생 결제 접근성 확보 | 결제 시작→완료 전환율, 해외 결제 성공률 |
| PRD-G-02 | 영상 콘텐츠 유출 방지 | 무단 다운로드/재배포 차단, 유출 발생 시 추적 가능 |
| PRD-G-03 | 다국어 학습 경험 제공 | 한/영(→중) 사용자의 자연스러운 탐색·결제·시청 |
| PRD-G-04 | 린한 운영 구조로 시작 | 초기 고정비 최소화, 매출 연동 변동비 중심 |

### 1.5 범위 제한 (Out of Scope)
MVP에서 **의도적으로 제외**:
- 라이브 실시간 클래스 — VOD 전용으로 시작
- **중국 본토 결제(알리페이·위챗) 및 방화벽(GFW) 대응 인프라** — 대만 검증 후 v1.1+
- 모바일 네이티브 앱 — 반응형 웹으로 대체
- **다중 강사 마켓플레이스** — 영구 제외(향후 계획 없음)
- **정식 DRM(Widevine/FairPlay)** — 월 유지비 최소화 위해 MVP 제외, v1.1+ 보류
- 수료증 발급, 커뮤니티/포럼
- 구독·번들 상품 — 단건 모델 검증 후 재검토

## Phase 2: Users

### 2.1 타깃 사용자
| ID | 유형 | 설명 | 핵심 니즈 |
|----|------|------|-----------|
| PRD-U-01 | 해외 수강생 (대만 우선·영어권) | 해외 거주, 한국 베이킹 콘텐츠에 관심 | 영어 UI, 막힘 없는 해외 결제, 안정적 영상 시청 |
| PRD-U-02 | 국내 수강생 | 한국 거주 베이킹 학습자 | 한국어 UI, 국내 간편결제, 모바일 시청 |
| PRD-U-03 | 운영자/강사 (단일 브랜드) | 클래스 콘텐츠 제작·판매 주체 | 영상·자료 업로드, 가격·공개 설정, 매출·수강 현황 파악 |

### 2.2 사용자 시나리오 (Use Cases)
**UC-01**: 해외 결제 — 관련 사용자: PRD-U-01
해외 방문자가 언어를 영어로 전환 → 클래스 상세·미리보기 확인 → 해외카드/PayPal로 결제 → 즉시 수강 권한 획득.

**UC-02**: 보안 시청 — 관련 사용자: PRD-U-01, PRD-U-02
수강생이 구매한 차시 영상을 재생 → 시청 위치(진도) 자동 저장 → 다운로드·우클릭 저장 시도는 차단되고 화면엔 식별 워터마크가 표시됨.

**UC-03**: 콘텐츠 등록 — 관련 사용자: PRD-U-03
운영자가 새 클래스 생성 → 차시별 영상 업로드 및 레시피 PDF 첨부 → 가격·통화·공개 여부 설정 → 게시.

**UC-04**: 국내 결제 — 관련 사용자: PRD-U-02
국내 수강생이 카카오페이/카드로 결제 → 서버 검증 통과 → 수강권 발급.

### 7.2 용어 정의 (Glossary)
- **VOD**: 사전 녹화 주문형 비디오
- **HLS**: HTTP 기반 적응형 스트리밍 방식
- **서명 재생 URL**: 단기 만료되는 인증 토큰이 포함된 영상 접근 URL (Mux JWT)
- **DRM**: 디지털 저작권 보호 (Widevine/FairPlay 등) — v1.1+ 예정
- **클라이언트 오버레이 워터마크**: 사용자 식별 정보를 영상 위에 실시간 표시해 유출원 추적하는 저비용 방식
- **MID**: 결제 가맹점 식별자
- **MoR**: Merchant of Record (Paddle 등)
- **RLS**: Supabase 행 수준 보안
- **i18n**: 다국어 국제화

### 7.3 참고 자료 및 변경 이력

**v1.0 → v1.1 변경 사항**:
- 비즈니스 모델 확정: 클래스 단건 구매
- 수강 기간 확정: 구매 후 영구 시청(만료 없음)
- 영상 호스팅 확정: Mux (DX, 향후 DRM 확장성)
- 영상 보안 확정: 서명 재생 URL(Mux JWT) + 클라이언트 오버레이 워터마크 / DRM 미적용(비용 최소화 위해 v1.1+ 보류)
- 운영자 범위 확정: 단일 브랜드, 추후 강사 추가 계획 없음
- 다중 강사 마켓플레이스 영구 제외, 정식 DRM MVP 제외

**기타 주의사항**:
- Mux는 DX와 확장성이 우수하나 기본 단가가 Cloudflare Stream보다 높음. 시청량 증가 시 전송비 모니터링 필요.
- 영상 완전 유출 차단은 기술적으로 불가능(화면 녹화). 목표는 '무단 재배포 억제 + 유출원 추적'.
